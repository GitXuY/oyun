Detailed Description

This module contains the user interface code. Oyun is a wxWidgets application. Its entry-point is handled by the OyunApp class, which creates a wxWizard containing all of our UI. All pages in the wizard are derived from the OyunWizardPage class.

The general flow of the Oyun wizard is as follows:

  • OyunWizard - the overall wizard object
  • PlayersPage - the user selects a list of players
  • TypePage - the user chooses whether they will run a Tournament or an EvoTournament
  • OneShotPage - if the user has chosen a one-shot round-robin tournament (the Tournament class), run it
  • OneShotFinishPage - save the data from a one-shot tournament
  • EvoPage - if the user has chosen an evolutionary tournament, (the EvoTournament class), run it
  • EvoFinishPage - save the data from an evolutionary tournament

Variable Documentation

wxEVT_ADD_PLAYER

Event fired when a player is added to the list.

The event itself is a wxNotifyEvent, and can be captured by a class with: 

        EVT_NOTIFY(wxEVT_ADD_PLAYER, wxID_ANY, Handler)

where Handler has the prototype: 

        void Handler(wxNotifyEvent &event);

The player pointer to be added may be recovered with: 

        Player *player = (Player *)event.GetClientData();
See also:
OyunWizard::AddPlayer
wxEVT_DATA_UPDATE

Event fired to signal generic "data" update.

This event is intended to trigger a general UI update by all wizard pages. Which data in particular has been updated is unspecified.

The event itself is a wxNotifyEvent, and can be captured by a class with: 

        EVT_NOTIFY(wxEVT_DATA_UPDATE, wxID_ANY, Handler)

where Handler has the prototype: 

        void Handler(wxNotifyEvent &event);
See also:
OyunWizard::Update
